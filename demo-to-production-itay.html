01

Reliability

ממשיך לעבוד גם כשדברים נופלים

Idempotency›

מה זה: אותה פעולה פעמיים = אותה תוצאה, בלי side effects.

באפליקציה שלך: INSERT INTO predictions … ON CONFLICT (user_id, game_id) DO UPDATE — משתמש לוחץ פעמיים, לא נוצרות שתי שורות.

Retry + Timeout›

מה זה: לנסות שוב כשקריאה נכשלת, ולוותר אחרי N שניות.

באפליקציה שלך: Groq API תקוע 30 שניות — retry אחד, ואז נכנסים ל-failed_summaries. לא נתלים לנצח.

Rate Limiting ★›

מה זה: לחסום יותר מ-N קריאות בפרק זמן — גם לקוחות שלך וגם אתה מול APIs חיצוניים.

באפליקציה שלך: Groq מגביל 30 req/min. אם ב-halftime אתה שולח 100 סיכומים במקביל, תקבל 429s. פתרון: queue + throttle.

Circuit Breaker ★›

מה זה: אחרי N כישלונות ברצף, לעצור קריאות ל-service ההוא ל-X דקות. מונע cascade failures.

באפליקציה שלך: API-Football נפל — במקום להקיש אותו 100 פעם בדקה (ועולה כסף), לעצור ל-5 דק, אז לנסות שוב.

Graceful Degradation›

מה זה: אם פיצ'ר לא-קריטי נופל, השאר ממשיך לעבוד.

באפליקציה שלך: nightly-summary נכשל → ה-leaderboard עדיין מתעדכן. המשתמש רואה "סיכומים בדרך" במקום דף שגיאה.

02

Observability

יודעים מה קורה בפנים

Logs (EF + DB)›

מה זה: טקסט מובנה שתיעד מה קרה מתי.

באפליקציה שלך: console.log ב-Edge Functions + pg_stat_statements ב-Supabase. לחפש אחורה "מה קרה ב-3 בבוקר כשהסיכום נפל".

Token tracking›

מה זה: ללוג את מספר הטוקנים של כל קריאת LLM.

באפליקציה שלך: input_tokens ו-output_tokens ב-ai_summaries. מאפשר לחשב $ לסיכום, ליום, לקבוצה.

Metrics dashboard ★›

מה זה: ויזואליזציה של מספרים מצטברים לאורך זמן.

באפליקציה שלך: ה-SQL שלנו (cost/day, failures, runs) → דף admin באפליקציה או Grafana. רואים טרנדים, לא רק נקודות.

Alerts›

מה זה: התראה אוטומטית כשמטריקה חורגת מרף.

באפליקציה שלך: cron: SELECT COUNT(*) FROM failed_summaries WHERE resolved=false — אם > 10, email אליך. יודע על בעיות לפני שמשתמשים מתלוננים.

Error tracking (Sentry)›

מה זה: service שתופס exceptions ומצרף קונטקסט (user, url, stack trace, browser).

באפליקציה שלך: React זורק ב-Picks page → Sentry מראה את השורה + הדפדפן + ה-state של הניחוש. מנצח console.error.

03

Security

מגנים על דאטה וגישה

Secrets (Vault) ★›

מה זה: אחסון מפתחות API מוצפן, נגיש רק בזמן ריצה.

באפליקציה שלך: Groq API key + API-Football key ב-Supabase Vault, לא ב-.env שנכנס ל-git.

HTTPS / SSL ★›

מה זה: הצפנת תעבורה בין דפדפן לשרת.

באפליקציה שלך: GitHub Pages נותן HTTPS אוטומטית. בלי זה, מישהו על WiFi יכול לצותת ל-login tokens של המשתמשים.

Auth + RLS›

מה זה: אימות משתמש + אכיפת גבולות דאטה ברמת ה-DB עצמה.

באפליקציה שלך: RLS policy SELECT predictions WHERE user_id = auth.uid() — משתמש א' לא יכול לראות ניחושים של משתמש ב', גם אם יעקוף את ה-frontend.

JWT ★›

מה זה: token חתום שמוכיח "המשתמש מאומת" בלי לגשת ל-DB בכל בקשה.

באפליקציה שלך: Supabase מנפיק JWT ב-login; frontend שולח בכל request header; EF מאמת את החתימה מקומית (מהיר).

Zero-Trust ★›

מה זה: להניח שכל בקשה עוינת, לאמת בכל שכבה. לא "סומכים כי זה פנימי".

באפליקציה שלך: EF לא סומך על user_id ב-request body — גוזר אותו מחדש מה-JWT ובודק RLS.

04

Deployability

מעלים ומחזירים בלי דרמה

Migrations›

מה זה: שינויי schema versioned — כל שינוי ב-DB הוא קובץ SQL.

באפליקציה שלך: 56 migrations. אם תפוצץ את ה-prod, אפשר להריץ אותן על DB חדש ולחזור למצב תקין.

Branches + PRs›

מה זה: לבודד פיצ'ר ב-branch, לסקור לפני merge.

באפליקציה שלך: feature/nightly-summary — מפתחים, push, PR, review, merge ל-main. main נשאר נקי. (המקום היחיד שעוד לא מושלם אחרי incident הauto-merge של Cursor.)

CI/CD ★›

מה זה: pipeline אוטומטי: push → tests → deploy.

באפליקציה שלך: GitHub Actions על push ל-main → מריץ tests → deploy ל-GitHub Pages. אתה רק pusher; deploy קורה לבד.

Docker ★›

מה זה: אריזת app + כל התלויות שלו ב-container שרץ בכל מקום.

באפליקציה שלך: לא קריטי — GitHub Pages + Supabase = serverless. ירלוונטי אם תארח בעצמך או תצטרך reproducibility של dev env.

ADR ★›

מה זה: Architecture Decision Record — מסמך קצר שמסביר "למה בחרנו X על פני Y".

באפליקציה שלך: "למה Groq ולא OpenAI?" — latency, cost, format. "You-עתידי" קורא ולא מבטל את ההחלטה בטעות.

05

Scalability

מחזיק עומס וגדילה

CDN ★›

מה זה: Content Delivery Network — עותקים של קבצים סטטיים במספר מיקומים גיאוגרפיים. משתמש מקבל מהקרוב.

באפליקציה שלך: GitHub Pages משתמש ב-Fastly CDN. משתמש בתל אביב טוען מ-edge תל אביב, לא מ-HQ של GitHub.

Connection pool›

מה זה: שימוש חוזר ב-DB connections במקום לפתוח אחד לכל request.

באפליקציה שלך: Supabase PgBouncer — 1000 משתמשים סימולטניים לא צריכים 1000 connections פתוחים; pool מנהל ~20.

DB Indexing ★›

מה זה: data structure שהופך queries ל-O(log n) במקום O(n).

באפליקציה שלך: index על predictions(game_id) — "כל הניחושים למשחק X" — DB קופץ ישר, לא סורק 10,000 שורות.

Query optimization›

מה זה: להאיץ SQL — joins חכמים, projections, הימנעות מ-N+1.

באפליקציה שלך: יש לך SELECT group_id FROM predictions WHERE user_id=X בלולאה = N+1. תיקון: JOIN אחד במקום.

Caching›

מה זה: שמירת תוצאות חישוב יקר, החזרת עותק על פנייה חוזרת.

באפליקציה שלך: leaderboard מחושב מחדש בכל page load → cache ב-Redis או materialized view ל-30 שניות, חישוב מחדש אחרי.

06

Durability

דאטה שורד אסונות

Backups (Supabase) ★›

מה זה: snapshots אוטומטיים של ה-DB, נשמרים N ימים.

באפליקציה שלך: Supabase PITR (Point-in-Time Recovery) — 7 ימים ב-free tier. אם הפלת טבלה בטעות, תוכל לשחזר.

Replication›

מה זה: עותק של DB מתעדכן בזמן אמת. failover אם ה-primary נופל.

באפליקציה שלך: Supabase מריץ replicas ברקע. גם אם primary קורס, replica קופץ פנימה.

DR plan›

מה זה: Disaster Recovery — מסמך כתוב: "אם DB נופל, עושים X, Y, Z".

באפליקציה שלך: "שחזר backup + עדכן Supabase URL בכל ה-EFs + הודע למשתמשים". אין לך כזה עדיין — כדאי לרשום 5 שורות לפני המונדיאל.

Data retention›

מה זה: כללים מה שומרים ומה מוחקים, ולכמה זמן.

באפליקציה שלך: "ניחושים — 2 שנים; auth logs — 90 יום". חלק מזה legal (GDPR), חלק עלויות אחסון.

DNS failover ★›

מה זה: אם שרת נופל, DNS מפנה משתמשים לשרת backup.

באפליקציה שלך: לא קריטי — GitHub Pages CDN מטפל. ירלוונטי רק אם תארח שרת בעצמך.

07

Cost & Governance

בר קיימא וחוקי

Token cost tracking›

מה זה: לרשום כמה עלתה כל קריאת LLM.

באפליקציה שלך: input_tokens * 0.00015 + output_tokens * 0.0006 לכל שורה — יודע עלות יומית, חודשית, ולכל קבוצה.

Budget alerts›

מה זה: התראה כשההוצאה חורגת מרף.

באפליקציה שלך: cron: SUM(cost) last 7 days — אם > $50, email. מונע runaway costs (פרומפט תקוע בלולאה).

Cost per user›

מה זה: unit economics — כמה עולה לך משתמש לחודש?

באפליקציה שלך: (Supabase $20 + LLM costs) / MAU. נניח $0.20/user. קריטי אם תרצה לגבות פעם.

Privacy / ToS›

מה זה: דפי Terms of Service ו-Privacy Policy.

באפליקציה שלך: חייב לציין מה אוסף (email, ניחושים), איך משתמש, עם מי משתף (Groq רואה דאטה משחקים). GDPR fine בלי.

Compliance›

מה זה: חוקים רגולטוריים (GDPR ל-EU, PCI לתשלומים).

באפליקציה שלך: משתמשים ב-EU → GDPR → חייב לאפשר מחיקת דאטה על בקשה. לא בעיה ברצינות עד 100 משתמשים.

זאת מפה, לא תפריט.
לחץ על כל מושג כדי לראות הסבר קצר ודוגמה קונקרטית מהאפליקציה שלך.
במפגש הבא: נעבור ביחד על הסימונים ונבחר מה חייב לפני מונדיאל ומה אחרי.
